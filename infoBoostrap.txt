PRÁCTICA 3 - USO DE FRAMEWORK BOOTSTRAP

ESTRATEGIA DE FICHEROS
Se han creado archivos independientes (*Bootstrap.html y estiloBootstrap.css) para no alterar el código original del proyecto. El archivo CSS nuevo recicla únicamente las variables de color, fuentes y elementos decorativos (estrellas, bordes premium), delegando toda la maquetación al sistema de Grid de Bootstrap 5.

USO DE GRID SYSTEM (RWD)
Se ha implementado la rejilla de Bootstrap para adaptar el layout a tres resoluciones:

Móvil (xs): Columnas apiladas (col-12) en formularios y listados de productos.

Tablet (md): Distribución de 2 columnas para noticias y sedes (col-md-6).

Escritorio (lg): Distribución de layout principal con barra lateral (col-lg-9 + col-lg-3) y footer a 4 columnas.

COMPONENTES BOOTSTRAP UTILIZADOS

A) Navbar (Barra de Navegación)
Ubicado en todos los ficheros HTML. Sustituye al menú manual CSS, ofreciendo comportamiento "responsive" (menú hamburguesa) automático en móviles.

B) Carousel (Carrusel de imágenes)
Ubicado en indexBootstrap.html. Sustituye las imágenes estáticas de bienvenida, permitiendo rotar contenido promocional.

C) Accordion (Acordeón)
Ubicado en indexBootstrap.html (Sección Consejos). Comprime la información vertical de la infografía original, mejorando la usabilidad en pantallas pequeñas.

D) Modals (Ventanas Modales)
Ubicado en productosBootstrap.html. Se utilizan para mostrar las reseñas de los productos sin recargar la página ni usar CSS :target complejo.

E) Cards & Badges
Uso extensivo de las tarjetas (Cards) para productos y noticias, y de las insignias (Badges) para etiquetas de "Oferta" o "Premium".
